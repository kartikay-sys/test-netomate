NetoMate Flow AI

A visual breakdown of the inference pipeline: from user request to AI-reasoned test flow suggestions.

Step 01

Client Payload

The UI extracts the user's working sequence from the textarea along with their natural language query, packaging it into a JSON bundle sent to the backend server.

JSON RequestLocal Network
Step 02

Preprocessing Intelligence

The backend analyzes textual data to uncover hidden query metadata. It maps out target categories (mobile/UI) and the desired testing phase (setup, teardown, validation).

Keyword MatchingIntent Extraction
Step 03

State Machine & Conflict Avoidance

Evaluates existing functions to determine precise execution state (e.g. MID_ACTION). It intelligently resolves any conflicts if the user asks for a UI step in a Mobile-only scope.

Category BoundsFlow State Logic
Step 04

Prompt Serialization

If the flow isn't empty, NetoMate serializes its entire 50-flow historical benchmark dataset into a single gigantic prompt text block, enriched by temporal logic hints.

Dataset InjectionTemporal Prompts
Step 05

LLM Engine Inference

The enriched prompt is fired to the Local LLM Model. The large language model contextualizes the injected historical logic to predict robust missing and next steps.

Local LLMJSON Constraints
Step 06

Pipeline Validation

LLM hallucinations are stripped. Outputs are audited against vocabulary, duplicates are killed, EPCEMS constraints enforced, and confidence limits are rigidly applied.

SecurityHallucination Filters
Step 07

Smart Display Balancing

A smart balancer evaluates the final suggestions list to guarantee that the UI gets both Intent-Aligned tools and Flow-Continuation defaults, ready to click and inject.

Output RoutingDOM UI Update
