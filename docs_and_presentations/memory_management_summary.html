What gets stored — silently, automatically
1
Query history
Every NLP query the engineer typed, in order. Used to resolve ambiguous follow-up queries like "what else" by referencing what was asked before.
2
Suggestion interaction log
For every suggestion shown — whether it was accepted, ignored, or rejected. Never re-suggests ignored functions for the rest of the session.
3
Flow edit history
Every version of the flow as it was built — including functions that were added and then removed. Enables undo/redo and drift detection.
When it gets injected into the prompt
Never automatically. Only when the engineer explicitly triggers it with a phrase like:
"based on what I did earlier" "continue from before" "what have I tested so far" "what did I miss earlier" "based on my history"
A compact distilled summary (~100 tokens max) is injected — not the raw log. Raw timestamps, ignored suggestions, and intermediate states never enter the prompt.
~100 tokens max distilled summary only raw log stays local

Hard boundary
Session memory influences suggestions but never overrides hard constraints. Category filter, state filter, and duplicate suppression always run after memory is applied. Memory is a hint to the LLM — not a rule.
memory = hint only constraints always enforced in code resets on session end
Storage
Entirely client-side. Python dict in memory keyed by session ID. No database needed. Every function add, query submit, and suggestion interaction appended in real time on the frontend. No backend call for logging.
client-side only session-scoped dict no DB required resets on refresh
