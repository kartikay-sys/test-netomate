ARCHITECTURE v2.0
Layer 0 — Input
⌨
Engineer Input
Two simultaneous inputs: (1) current_flow — partial sequence of functions built so far  ·  (2) nlp_query — natural language intent string
current_flow: list[str] nlp_query: str step_count: int category: UI | Mobile | Hybrid
Layer 1 — Intent Classifier
⚡
Rule-based Intent + Category Detector
Keyword scan on NLP query to detect intent type. Prefix scan on current_flow to detect category (epcems_ → UI, mobile fns → Mobile, both → Hybrid). Routes to one of three engine paths. Zero ML needed here — pure string matching.
step_count == 0 → cold start "next/complete/missing" → probabilistic "improve/validate/robust" → reasoning category injected into all paths
Layer 2 — Core Engine (no hardcode · no heavy LLM)
◎
E1 — Start Predictor
Replaces hardcoded defaults.

Scans all 50 flows, counts first-step frequency distribution. Returns top-k by count.

search_pcap_file → 22×
epcems_login → 12×
preset_mobiles → 4×

Auto-updates when dataset grows. Zero hardcoding.
triggered: step_count == 0 first_step_freq table collections.Counter
⇢
E2 — Markov Chain Engine
Upgraded from raw n-gram. Stores transition probabilities learned from all flows.

P(packet_capture_stop | packet_capture_start) = 0.91

Order-2 preferred. Backoff strategy when no match:
trigram → bigram → unigram

Laplace smoothing prevents zero probabilities on unseen transitions.
triggered: "next/complete" order-2 markov laplace smoothing backoff strategy
≈
E3 — Embedding Retriever
Converts each flow into a dense vector using sentence-transformers (MiniLM-L6).

Input flow embedded → cosine similarity against all 50 flow vectors → top-k similar flows retrieved → next steps extracted from those flows.

Handles unseen flow patterns. Works with small data.
MiniLM-L6-v2 cosine similarity sentence-transformers FAISS / numpy
↓
↓
↓
⊕   Weighted Score Combiner
score(fn) = α × frequency_score  +  β × similarity_score  +  γ × user_click_weight   |   top_3 = argsort(score)[:3]
Layer 3 — Micro-LLM (optional · <10% queries)
◈
Lightweight Reasoning Layer — Gemini Flash
Not used for generation. Used only for re-ranking + explanation. Engine produces top-5 candidates via Markov + Embeddings. LLM receives: current_flow + top-5 candidates + 3 most similar flows from dataset. Task: rerank candidates and generate one-line explanation per suggestion. Prompt is tiny — no hallucination risk.
triggered: abstract intent only re-rank only · not generate Gemini 2.5 Flash injected context from dataset <10% of all queries
Layer 4 — Feedback Learning Loop
↻
User Click Tracker → Weight Updater
Tracks which suggestion the engineer actually selected. Updates transition weights over time. System gets smarter with every use without retraining.
weight(fn) += δ  ·  click_event
stored: (flow_state, suggestion, selected)
Layer 5 — Output
✦
Top 3 Suggestions
All paths converge here. Each suggestion carries function name, confidence score, explanation string, occurrence count, and source path (markov / embedding / llm).
fn_name: str confidence: 0–100 explanation: str occurrences: int source: markov | embed | llm
